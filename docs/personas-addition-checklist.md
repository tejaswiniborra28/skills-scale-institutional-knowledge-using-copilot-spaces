# Personas Addition Checklist

Use this checklist when proposing or adding new personas to docs/octoacme-roles-and-personas.md.

- Title: [Name of persona]
- Short summary (1-2 sentences)
- Responsibilities (bullet list)
- Interacts with: [list of primary touchpoints]
- Suggested owner (name / role)
- Acceptance criteria:
  - [ ] Content aligns with existing process docs
  - [ ] Update improves clarity or closes a documented gap
  - [ ] Proposed content reviewed by stakeholders (list reviewers)
  - [ ] Documentation published to docs/ and linked in project README
- Reviewers to request:
  - Product Manager: @<pdM-gh-username>
  - Project Manager: @<pm-gh-username>
  - Engineering Lead / Architect: @<tech-architect-gh-username>
  - QA or Release Engineer (if applicable): @<qa-gh-username>
- Notes / Rationale:
  - [Explain why the persona is needed and what gap it fills]
- Link to PR / Issue:
  - Issue: #4
  - PR: (add PR link when available)
